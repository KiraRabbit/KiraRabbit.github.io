Java常用注解（不定期更新）
发表于2020-08-30|更新于2020-08-30
|阅读量:

@ResponseBody

response body 中，一般在异步获取数据时使用【也就是AJAX】，在使用 @RequestMapping后，返回值通常解析为跳转路径，但是加上 @ResponseBody 后返回结果不会被解析为跳转路径，而是直接写入 HTTP response body 中。 比如异步获取 json 数据，加上 @ResponseBody 后，会直接返回 json 数据。
作用:把方法的返回值转换为json返回

@Component spring扫描这个组件,将这个注解的类初始化为bean,纳入到ioc的容器里

controller 控制层
service 服务层
dao 数据访问层
当一个类的角色不知道是哪一层的时候,就使用@Component

对象只要在ioc的容器中,在任何地方都可以用AutoWired注入
加入@Component,这样这个类就自动加入了spring里面去了

@ConfigurationProperties

@ConfigurationProperties 将配置文件的属性和当前类的成员变量进行绑定,当前这个注解不会初始化当前类对象

@RestController

包括了@Controller,以及@ResponseBody等注解

@Bean

初始化一个方法返回值类型的bean对象,bean的id就是方法的方法名
相当于之前spring的配置文件中的bean标签===>

@Configuration:

告诉spring当前是一个配置类
相当于标签

@PathVariable

controller层

用来写

@ mapperScan

可以指定要扫描的Mapper类的包的路径

@EnableDiscoveryClient

@EnableDiscoveryClient放在启动类中
在启动类中开启Eureka客户端功能

组合注解@SpringCloudApplication

@SpringCloudApplication包含3个注解
@SpringBootApplication ====>SpringBoot注解
@EnableDiscoveryClient ====>注册服务中心Eureka注解
@EnableCircuitBreaker ====>熔断器注解

对于SpringCloud来说，这是每一微服务必须应有的三个注解，所以才推出了@SpringCloudApplication这一注解集合。

@HystrixCommand

@HystrixCommand(fallbackMethod = “queryByIdFallBack”)：用来声明一个降级逻辑的方法

Hystix的默认超时时长为1:请求超过1s,会返回错误信息,可以通过配置修改这个值

这个配置会作用于全局所有方法。

@DefaultProperties

@DefaultProperties(defaultFallback = “defaultFallBack”)：在类上指明统一的失败降级方法,此时只需要写@HystrixCommand就可以了
